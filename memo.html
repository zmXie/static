小程序审核被拒

被拒 1

登录报错提示：“登录失败，用户数据异常，用户id:xxxxxx”

原因： 一个用户不能绑定两个微信（也就是不能绑定两个openid）。比如在微信A上登录用户A（此时用户A已绑定微信A的openid），那就不能在微信B上登录用户A，这是我们小程序登录的通用逻辑，都会存在这个问题。
解决方案：申诉，选择“我不接受”，内容如下：
“你好，登录报错属于我们正常的业务需求，我们为了确保用户的唯一性，一个用户不能在两个微信上登录，否则抛出异常，而通常情况下不会出现这种问题。所以请审核员在对应微信上登录该账号，或者换一个账号登录。”
【openid:一个微信用户对应一个公众平台应用(公众号/小程序)的唯一标识；unionid：一个微信用户对应一个开放平台账号的唯一标识(开放平台可以绑定同主体的小程序/公众号，每个应用的unionid相同，这样就可以打通应用数据)】
被拒 2

提示：“小程序账号登录功能暂未符合规范要求，请在用户了解体验小程序功能后，再要求用户进行账号登录”

原因：需要登录才能进入小程序，导致体验受阻。
解决方案：做游客模式，把登录后置，必要的功能模块再弹框提示用户登录（弹框必须支持取消）。
你好，我们小程序默认支持游客模式，在进入首页、健康测量、健康科普等页面都不需要登录，只会在业务必要时才提示用户登录，用户也可以自由选择是否登录，谢谢。
被拒 3

提示“你的小程序涉及提供用户随时随地自定义生成的内容…，请补充选择：社交-笔记类目。”

原因：用户发布社区内容，需要添加社交笔记类目。
解决方案：添加社交-笔记类目，再提交审核，以后如果有其他小程序涉及到该功能，要提前添加该类目，避免卡审核时间。
被拒 4

提示“涉及平台未允许的内容：匿名社交。”

原因：发布社区内容时，可以勾选是否匿名发布
解决方案：先去掉匿名发布选项，避一避风头再放出来，或者做一个审核配置，审核期间把相关功能关闭，审核过了再放出来。
被拒 5

提示：“你的小程序【评论】功能在进行内容安全验证时，存在信息安全风险，请尽快完善内容机制…”

原因：未接入微信内容安全API，且发布之后直接展示出来了。
解决方案：接入微信内容安全API，或者申诉，选择“我不接受”，内容如下：
“我们有内容审核机制，用户发布的所有信息都会经过后台审核，确保没有违规信息才会真实发布，审核完成前，只有用户自己可以看到，安全审核完成前，只有用户自己可以看到，其他用户无法看到，请审核员发布内容后，再登录另一个账号检查。”
被拒 6

提示：“你好，你的小程序涉及提供播放、观看等服务，请补充选择：文娱-其他视频类目。”

原因：「文娱-其他视频」类目过期，重新申请需要视听许可证，该证书目前申请困难。
解决方案：申诉，弱化视频功能，内容如下：
“小程序主要功能是帮助患者做健康管理，其中有一个甲氨蝶呤操作视频播放，该视频仅用于介绍产品如何使用，为了支持患者管理健康，不属于文娱类功能，且不是主要内容，我们认为所选类目与服务内容一致，请审核员酌情考虑，谢谢。”
（上传小程序页面图片，证明视频播放不是主要功能）

功能及类目

1. 包含资讯相关功能，需要添加“资讯-信息资讯”类目
2.包含video组件，需要添加“文娱-其他视频”类目（不强制绑定该类目，但是审核经常会被拒，可以申诉，例如被拒6），或者使用腾讯视频插件（需要将视频传到腾讯视频，有广告）
3. 包含社区发帖评论功能，需要添加“社交-笔记”类目
4. 包含地图导航功能，需要添加“交通服务-地图导航”类目

小程序直播

1. 使用小程序推流拉流组件：live-pusher，live-player； 需要在开发管理中开通接口（必须有直播类目，申请类目需要相关资质）；使用自己的流媒体服务器。
2.使用小程序直播插件：live-player-plugin；需要在小程序后台开通直播（部分类目可开通，且小程序近90天存在支付行为），使用腾讯云服务器。具体操作：在小程序后台创建直播间，微信扫码即可开播（打开直播插件），小程序端通过wx.navigateTo({url: `plugin-private://wx2b03c6e691cd7370/pages/live-player-plugin?room_id=${roomId}`}) 打开插件观看直播（room_id通过api获取小程序后台创建的直播间id）。
3. 小程序内嵌视频号直播组件：channel-live； 小程序主体需要与视频号主体一致。
4. 小程序跳转视频号，通过视频号id跳转到视频号直播，不要求主体一致。
5. 如果只做拉流，可以用video组件，并设置is-live属性，基础库版本（2.28.1以上）。
6. 如果只做拉流，也可以使用webview嵌入网页直播流，需要设置业务域名。

小程序订阅消息

订阅消息是给订阅者推送消息，谁订阅了就给谁发送消息，同时后端可以指定推送到体验版还是正式版。
1. 在小程序后台创建订阅模版（分为一次性订阅：订阅一次可以发一次消息；和长期订阅：订阅一次可以无限发消息，特定类目才可以使用），会生成模版id。
2. 前端小程序调用requestSubscribeMessage，传入模版id，申请订阅授权。
3. 后端调用微信接口发送订阅消息，传入模版id、openid（接受者）、page（页面跳转路径）、miniprogram_state（小程序版本，体验版还是正式版）
4. 用户收到订阅消息后，点击跳转对应page。

绑定与关联规则

小程序绑定开放平台

一个小程序只能绑定一个开放平台，一个开放平台最多能绑定50个同主体小程序、5个不同主体小程序（公众号同样如此）。 同一个微信用户，在开放平台绑定的不同应用中，UnionID相同，OpenId不相同。

公众号关联小程序

关联小程序之后，可以从公众号跳转小程序，也可以从小程序跳转公众号，也可以在小程序中通过webview打开公众号文章（3.4.8版本之后，可以通过wx.openOfficialAccountArticle 打开任意公众号文章）。
一个公众号可关联10个同主体，3个非同主体小程序。

钉钉消息跳转配置

1. 跳转患者详情用药信息页签 

HOST/mobile/xg-scp-dynamic-app-m/#/routing/MTT_DD_PATIENT_DETAIL?corpid=corpid&agentid=agentid&loginType=DD&patientId=patientId&tabIndex=1
参数说明：
• HOST: 环境域（例如：https://test-m.szyun.info）
• corpid: 钉钉的企业ID
• agentid: 钉钉的应用ID
• patientId: 患者ID

2. 跳转领药反馈详情

HOST/mobile/xg-scp-dynamic-app-m/#/routing/MTT_DD_FEEDBACK_DETAIL?corpid=corpid&agentid=agentid&loginType=DD&targetId=targetId
参数说明：
• HOST: 环境域
• targetId: 反馈单ID

3. 维图可患者详情-调研问卷页签 

HOST/mobile/xg-scp-dynamic-app-m/#/routing/WTK_DD_PATIENT_DETAIL_RESEARCH?corpid=corpid&agentid=agentid&loginType=DD&patientId=patientId&tabType=RESEARCH
参数说明：
• HOST: 环境域
• corpid: 钉钉的企业ID
• agentid: 钉钉的应用ID
• patientId: 患者ID

Git 常用命令备忘

1. 浅克隆与分支获取

浅克隆（depth=1）：
git clone --depth=1 http://xxx.git
# 示例
git clone --depth 1 http://192.168.3.200/ios/PatientCircle.git
浅克隆后拉取其他分支：
git remote set-branches origin 'branch_name'
git fetch --depth 1 origin branch_name
git checkout branch_name

2. 撤销与重置操作

丢弃本地所有内容，以远程为准：
git fetch --all
git reset --hard origin/master
常见撤销命令：
• 撤销工作区所有修改：git checkout -- .
• 删除未追踪文件：git clean -d -f
• 撤销暂存区内容（提交）：git reset --mixed HEAD

3. 特定提交与合并

应用特定提交： git cherry-pick <commit-hash>
不切换分支合并（合并到测试分支）：
# 1. 拉取测试分支最新代码
git fetch origin release-test:release-test
# 2. 合并当前分支到测试分支
git merge current-branch release-test
# 3. 推送测试分支
git push origin release-test

4. 文件追踪与忽略操作

临时忽略已追踪文件：
git update-index --assume-unchanged filename
恢复追踪：
git update-index --no-assume-unchanged filename
临时储藏：
git stash / stash apply / stash pop

企微自建应用配置指南

1. 创建应用与基础凭证

进入企微管理后台：企微应用管理入口
创建应用后将获得：
• AgentId (即 AppID)
• Secret (秘钥，后端鉴权调用接口必需)

2. 授权域名与 JS-SDK 配置

2.1 域名校验： 下载校验文件放入可信域名根目录，确保 http://domain.com/WW_verify_xxx.txt 可访问。
2.2 网页授权域名： 只有配置了可信域名，回调获取 code 后才能正常跳转打开项目（用于自定义登录）。
2.3 JS-SDK 可信域名： 调用企微 JS 接口（如分享、选人等）必须在可信域名下。
⚠️ 重要提示（ICP 备案限制）：
可信域名必须是企业 ICP 备案域名或其子域名。若实际部署环境在不同域名（如 szyun.info），需运维通过 Nginx 将备案域名（如 mediportal.com.cn）映射到实际环境。

3. 聊天工具栏与 URL 规则

配置示例：
http://testwx-ms.mediportal.com.cn/scp-basic/wxwork/public/ww55657e646836968a/1000008/routing?scope=snsapi_base&redirect=http://testwx-m.mediportal.com.cn/mobile/xg-scp-dynamic-app-m/#/wechatLogin/WORK_WX_PATIENT_DETAIL?scope=snsapi_privateinfo&loginType=WXWORKH5&...&corpid=ww55657e646836968a&agentid=1000008
参数解析：
• scope=snsapi_base: 静默授权（仅获取 code，不弹窗）
• scope=snsapi_privateinfo: 用户授权（可获取头像、二维码，需用户同意）
• WORK_WX_USER_ID: 企微用户 ID（内置变量）
• WORK_WX_PATIENT_DETAIL: 预设的内置跳转路由变量

4. 授权重定向流程

访问 routing 接口后，系统会重定向至企微官方授权页：
https://open.weixin.qq.com/connect/oauth2/authorize?appid=ww55657e...&redirect_uri=...&response_type=code&scope=snsapi_base&state=...&agentid=1000008
